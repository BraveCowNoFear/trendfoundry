# TrendFoundry Design System

## Design Direction

TrendFoundry should feel like a quiet intelligence product, not a loud AI tool directory. The interface should help a buyer scan, trust, and act.

## Principles

- Minimal but not empty: every visible element must clarify source, fit, freshness, or next action.
- Editorial confidence: use restrained typography, clear titles, and factual labels.
- No hype decoration: avoid gradients, decorative blobs, fake analytics, and inflated claims.
- Visuals must carry information: charts, source-mix bars, tables, and previews are preferred over decorative imagery.
- Dense where useful: cards can carry detailed production inputs, but spacing and hierarchy must make scanning easy.
- Trust over excitement: show stale/cache state and source errors plainly.

## Visual Tokens

- Ink: `#15171a`
- Muted: `#666d75`
- Paper: `#f7f8fa`
- Panel: `#ffffff`
- Line: `#dfe3e8`
- Accent: `#0f6b5f`
- Warm accent: `#8a5a18`

## Component Rules

- Cards: radius `8px`, no nested cards, one clear border.
- Buttons: rectangular, `6px` radius, text only for commands where iconography would not clarify intent.
- Filters: compact segmented controls with clear active state.
- Status labels: small uppercase tracking, never decorative pills without operational meaning.
- Mobile: single-column, no horizontal overflow, actions wrap cleanly.

## Copy Tone

- Use precise language: source-backed, recordable, cached fallback, sample order.
- Avoid guaranteed income, guaranteed views, or platform-growth promises.
- Make the next buyer action visible within the first viewport.
