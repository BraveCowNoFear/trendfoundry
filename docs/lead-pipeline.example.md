# TrendFoundry Lead Pipeline

This is the committed example. The real pipeline is generated by:

```bash
npm run leads
```

Generated files:

- `data/leads.json`
- `docs/lead-pipeline.md`

Both generated files are ignored by Git because they may contain buyer contact details.

## Stage Model

- `untriaged`: matching issue exists but has no lead labels.
- `new_lead`: issue is labeled `lead` or `sample-pack`.
- `qualified`: issue is labeled `qualified`.
- `paid`: issue is labeled `paid`.
- `not_fit`: issue is labeled `not-fit`.
- `closed`: issue is closed without a stronger conversion label.

## Operating Loop

1. Run `npm run leads`.
2. Open `docs/lead-pipeline.md`.
3. Validate every new channel or team.
4. Reply on GitHub or email with the current sample-pack path and payment instructions.
5. Update GitHub labels so the next sync reflects the latest stage.
